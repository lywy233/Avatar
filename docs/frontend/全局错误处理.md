# React 全局错误处理与弹窗

## 为什么需要全局错误处理

在 React 应用中，API 错误可能在多个层级被捕获：

1. **组件内部** - try-catch 捕获同步错误
2. **Promise rejection** - `.catch()` 捕获异步错误
3. **React 渲染层** - Error Boundary 捕获渲染错误
4. **全局事件** - `unhandledrejection` 和 `error` 事件捕获遗漏的错误

`@agentscope-ai/chat` 组件内部有自己的错误处理逻辑，当 API 返回 502 等错误时，组件内部会捕获并消化这些错误，导致用户看不到任何提示。因此需要全局错误处理机制作为"安全网"。

## 推荐的全局错误处理方式

### 1. Error Handler Provider + Hook

创建 `use-error-handler.tsx` hook，提供全局错误状态管理和弹窗控制：

```tsx
// hooks/use-error-handler.tsx
import { createContext, useContext, useState, useCallback, useEffect, type ReactNode } from 'react'
import {
  AlertDialog,
  AlertDialogContent,
  AlertDialogDescription,
  AlertDialogHeader,
  AlertDialogTitle,
} from '@/components/ui/alert-dialog'

interface ErrorState {
  isOpen: boolean
  title: string
  message: string
}

interface ErrorContextValue {
  showError: (title: string, message: string) => void
}

const ErrorContext = createContext<ErrorContextValue | null>(null)

export function useErrorHandler() {
  const context = useContext(ErrorContext)
  if (!context) {
    throw new Error('useErrorHandler must be used within an ErrorHandlerProvider')
  }
  return context
}

export function ErrorHandlerProvider({ children }: { children: ReactNode }) {
  const [error, setError] = useState<ErrorState>({
    isOpen: false,
    title: 'Error',
    message: '',
  })

  const showError = useCallback((title: string, message: string) => {
    setError({ isOpen: true, title, message })
  }, [])

  const handleClose = useCallback(() => {
    setError((prev) => ({ ...prev, isOpen: false }))
  }, [])

  // 监听全局未处理的 Promise 错误和 JS 错误
  useEffect(() => {
    const handleUnhandledRejection = (event: PromiseRejectionEvent) => {
      const error = event.reason
      const message = error instanceof Error ? error.message : String(error)
      // 忽略取消请求的错误
      if (message && !message.includes('AbortError') && !message.includes('aborted')) {
        showError('Unexpected Error', message)
      }
    }

    const handleGlobalError = (event: ErrorEvent) => {
      const message = event.message
      // 忽略常见的非关键错误
      if (message && !message.includes('ResizeObserver') && !message.includes('Warning')) {
        showError('Unexpected Error', message)
      }
    }

    window.addEventListener('unhandledrejection', handleUnhandledRejection)
    window.addEventListener('error', handleGlobalError)

    return () => {
      window.removeEventListener('unhandledrejection', handleUnhandledRejection)
      window.removeEventListener('error', handleGlobalError)
    }
  }, [showError])

  return (
    <ErrorContext.Provider value={{ showError }}>
      {children}
      <AlertDialog open={error.isOpen} onOpenChange={handleClose}>
        <AlertDialogContent>
          <AlertDialogHeader>
            <AlertDialogTitle>{error.title}</AlertDialogTitle>
            <AlertDialogDescription>{error.message}</AlertDialogDescription>
          </AlertDialogHeader>
        </AlertDialogContent>
      </AlertDialog>
    </ErrorContext.Provider>
  )
}
```

### 2. 在应用中全局注册 Provider

在 `main.tsx` 或 `App.tsx` 中全局引入：

```tsx
import { ErrorHandlerProvider } from '@/hooks/use-error-handler'

function App() {
  return (
    <ErrorHandlerProvider>
      {/* 你的应用内容 */}
    </ErrorHandlerProvider>
  )
}
```

### 3. 在需要的地方调用 showError

```tsx
import { useErrorHandler } from '@/hooks/use-error-handler'

function MyComponent() {
  const { showError } = useErrorHandler()

  const handleApiCall = async () => {
    try {
      const response = await fetch('/api/data')
      if (!response.ok) {
        showError('API Error', `Request failed: ${response.status}`)
        return
      }
      // 处理响应
    } catch (error) {
      showError('Network Error', error instanceof Error ? error.message : 'Unknown error')
    }
  }

  return <button onClick={handleApiCall}>Submit</button>
}
```

## 为什么选择 AlertDialog 而不是 message/toast

| 特性 | AlertDialog | message/toast |
|------|-------------|---------------|
| 交互 | 必须点击确认 | 自动消失 |
| 打断性 | 强制用户看到 | 可能被忽略 |
| 适用场景 | 错误、确认操作 | 操作反馈 |
| 可访问性 | 更好的焦点管理 | 通常无障碍支持较弱 |

对于 API 错误和意外情况，AlertDialog 更合适，因为：
- 用户必须处理错误才能继续操作
- 错误信息更醒目，不容易被忽略
- 更好的可访问性支持

## 三层错误处理策略

推荐实现三层错误处理：

1. **组件内 try-catch** - 处理同步和基本的异步错误
2. **API 层包装** - 在 fetch/axios 层统一处理 HTTP 错误
3. **全局事件监听** - 作为安全网，捕获所有遗漏的错误

```tsx
// API 层包装示例
const apiFetch = async (url: string, options?: RequestInit) => {
  try {
    const response = await fetch(url, options)
    
    if (!response.ok) {
      const errorData = await response.json().catch(() => ({}))
      throw new Error(errorData.message || `HTTP ${response.status}`)
    }
    
    return response
  } catch (error) {
    // 重新抛出，让调用方处理
    throw error
  }
}
```

## 注意事项

1. **不要在 Error Handler 中再次抛出错误** - 会导致无限循环
2. **过滤无害错误** - 如 AbortError、ResizeObserver 警告
3. **考虑用户体验** - 错误信息要友好，不要暴露内部细节
4. **日志记录** - 生产环境应该上报错误到监控服务
