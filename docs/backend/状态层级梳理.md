# 关于状态层级

## 应用设置
应用启动时确定，为全局的pydantic settings

## 用户设置
用户的设置信息，通过用户的 id 可以获取
pydantic类型读取

## 智能体设置
智能体的相关设置信息，默认为一个json 通过pydantic类型读取

## 运行时配置
运行时配置由每次输入决定，将会通过每次输入得到具体的

## 运行时状态：
基于当次输入和应用逻辑联合确定，于得到当次输入时确定

# 总结
全局 下有多个用户

用户 下有多个智能体

智能体有自身的配置

# 额外
agent teams
额外的配置，独立于用户等，用于管理多个智能体之间的行为逻辑
为独立配置项


# 代码风格: 
非必要不要进行json配置外的操作，如果有必要性，则修改读取和写入逻辑，使得对应结构类型能使用这种方式实现
